# 01 · 安装笔记

在 conda `py311_leisaac` 里装 RLinf。

## 思路

走方案 B：`export VIRTUAL_ENV=$CONDA_PREFIX` 让 uv 用 conda env 作目标，复用已装的 LeRobot/SmolVLA。代价是自己解依赖冲突。

`pyproject.toml` 的 `[tool.uv].override-dependencies` 锁了核心包：

| 包 | 版本 |
|---|---|
| `torch / torchvision / torchaudio` | 2.6.0 / 0.21.0 / 2.6.0 |
| `torchcodec` | 0.2 |
| `transformers` | `<=4.57.6`（v5 重命名了 `AutoModelForVision2Seq`，会 ImportError） |
| `datasets` | 3.6.0 |
| `xgrammar` | 0.1.19 |

extras 互斥：`embodied ⊥ agentic-sglang ⊥ agentic-vllm ⊥ franka ⊥ ...`。装了 embodied 就别再装 agentic。

SmolVLA 不在 `install.sh --model` 列表里，按 [AGENTS.md](../../AGENTS.md) 的 "New embodied model" 自己接。

## 安装命令

```bash
conda activate py311_leisaac
export VIRTUAL_ENV="$CONDA_PREFIX"
cd /home/zzg/workspace/pycharm/RLinf

uv sync --active --extra embodied --no-install-project
uv pip install -e .
```

**不要** `bash requirements/install.sh agentic`（是给 LLM RL 用的，跟 embodied 冲突）。

验证：

```bash
python -c "import rlinf; print(rlinf.__file__)"
python tests/unit_tests/check_import_rlinf_package.py --workers 8
```

预期会剩这两个 FAILED（agentic 栈，可忽略）：
- `rlinf.workers.rollout.vllm.vllm_worker`
- `rlinf.workers.sft.megatron_vlm_sft_worker`

## 遇到的坑

**transformers 5.x → 4.57.x**：LeRobot 带的是 5.8.1，`uv sync` 没强降。

```bash
pip install "transformers>=4.57.1,<=4.57.6" "tokenizers>=0.22,<0.23"
```

如果 SmolVLA 因 transformers 降级加载失败，只能切方案 A（独立 venv，见底部）。

**`latex2sympy2` 缺失**：`rlinf/algorithms/rewards/__init__.py:16` 顶层 import math reward 没 try/except，导致 VLA 也加载不了：

```bash
pip install rlinf_latex2sympy2
```

**Ray 启动失败：虚拟网卡 `198.18.0.1`**。`198.18.0.0/15` 是 IANA 保留段，90% 是 ClashX / V2Ray / mihomo 等 TUN 模式造的。单机绕过：

```bash
ray stop --force
ray start --head --node-ip-address=127.0.0.1 --port=6380 \
  --num-cpus=2 --num-gpus=0 --disable-usage-stats
export RAY_ADDRESS=127.0.0.1:6380
```

多机不能用 `127.0.0.1`，要换物理网卡 IP；根治是关代理 TUN。

## 后备方案 A（独立 venv）

如果方案 B 冲突太多：

```bash
conda deactivate
bash requirements/install.sh embodied \
  --venv .venv-rlinf --python 3.11.14 --install-rlinf
```

独立 venv 训 RL，conda env 留给 SmolVLA 推理，ckpt 跨环境用文件传。
