# 10 · 未来改进方向

按预期 ROI 排序。每条标注收益 / 成本 / 判定。

## 训练速度

| # | 改进 | 收益 | 成本 | 判定 |
|---|---|---|---|---|
| 1 | **SGLang 替换 HF 推理** | 3-5×（CUDA graph + paged KV） | 大工程，SmolVLA flow 不是 token-AR，要自写 runner | 长期跑才上；换 OpenVLA / π0 主流模型时再上 |
| 2 | **rollout/actor 分卡** | 2× 吞吐（并行） | 2 张 24G+ 卡 | 硬件到位就上，零代码 |
| 3 | **增大 `total_num_envs`** | 1.5-2×（摊薄渲染） | 显存。base 冻结后 actor ~12G，env+sim ~6G | 跑稳直接试 16，OOM 回退 |
| 4 | **激进 gradient ckpt** | 5-10% 显存 | -10% 速度 | OOM 时上，否则不动 |
| 5 | **减小 `action_chunk_size`** | 单 chunk attn `O(chunk²)`，砍半省 75% FLOPs | 必须重 SFT，长视距可能掉精度 | 除非 30FPS 用不满 50 否则不动 |
| 6 | **`torch.compile`** | 10-20% | dynamic shape 兼容；LoRA+FSDP+compile bug 多 | 先稳定再优化 |

## 模型精度

| # | 改进 | 收益 | 成本 | 判定 |
|---|---|---|---|---|
| 1 | **增加 LoRA rank** | 精度上限 ↑ | rank 32→64：1.5M→3M，显存基本不变 | reward 长期不涨先提到 64 |
| 2 | **`critic_warmup_steps > 0`** | 早期 advantage 更准 | 前 N 步"白跑" rollout | 前 100 步 value loss 剧烈震荡上 50 |
| 3 | **reward shaping** | +20% 成功率可能 | 改了历史曲线不可比 | 见 [12](12-调参指南_提高任务成功率.md) |
| 4 | **Curriculum / Hindsight** | 加速收敛 | 改 env reset 逻辑 | 100 epoch reach < 30% 再上 |
| 5 | **混合 SFT 数据继续训** | 避免 RL 漂离 motion prior | 接 SFT dataloader 每 N 步插 SFT step | RL 后期动作变形（瞎抖 / 撞 base）强烈建议 |
| 6 | **真机 fine-tune（sim2real）** | 跨越 sim2real gap | DR + 真机几条 SFT alignment | Phase 6 |

## 工程改进

1. **checkpoint 自动恢复**：定期 LoRA + optimizer state 落盘，启动检测断点
2. **eval 自动化**：`val_check_interval: 20` 时同时录 video + 上传 W&B
3. **多 seed 平均**：PPO 方差大，单 seed 不可比，3 seed 并行
4. **更详细 metric**：per-stage reward 拆分（reach / grasp / move / on_plate / success 分开记），现在只看总 reward 看不出卡在哪
