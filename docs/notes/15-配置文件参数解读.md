# 15 · 配置文件参数解读

针对 [`maniskill_ppo_smolvla_so101.yaml`](../../examples/embodiment/config/maniskill_ppo_smolvla_so101.yaml)。

## 文件结构

```yaml
defaults:      # Hydra 拼接其它 yaml
hydra:         # Hydra 框架设置
cluster:       # Ray 集群和 GPU 分配
runner:        # 主循环
algorithm:     # PPO 算法（loss / advantage / clip）
env:           # 仿真环境（train + eval）
rollout:       # 推理端
actor:         # 训练端
reward:        # reward model（这里不用）
critic:        # 独立 critic（这里不用，value head 内嵌）
```

## defaults —— 文件拼接

```yaml
defaults:
  - env/maniskill_so101_pick_place@env.train
  - env/maniskill_so101_pick_place@env.eval
  - model/smolvla@actor.model
  - training_backend/fsdp@actor.fsdp_config
  - weight_syncer/patch_syncer@weight_syncer
  - override hydra/job_logging: stdout
```

`A@B` 把 `A.yaml` 塞到当前文件 `B` 字段下。`env/...@env.train` 复制一份到 train，再复制到 eval；后面在 `env.eval` 覆盖差异化字段。

## hydra

```yaml
hydra:
  run.dir: .                          # 不创建 outputs/<date> 子目录
  output_subdir: null
  searchpath:
    - file://${oc.env:EMBODIED_PATH}/config/
```

`EMBODIED_PATH` 由 `run_embodiment.sh` 设置，指向 `examples/embodiment/`。

## cluster

```yaml
cluster:
  num_nodes: 1
  component_placement:
    actor,env,rollout: 0-0     # 节点 0 GPU 0
```

`0-0` = 节点 id - GPU id。两卡分卡：`actor: 0-0, env: 0-0, rollout: 0-1`。8 卡 FSDP：`actor: 0-0,0-1,...,0-7`。

## algorithm —— PPO 灵魂

### 骨架

```yaml
adv_type: gae               # GAE
loss_type: actor_critic     # actor + value
normalize_advantages: True
kl_penalty: kl
group_size: 1               # PPO 不分组（GRPO 用）
reward_coef: 1.0
```

`adv_type / loss_type` 决定走 PPO 不 GRPO/Reinforce，由 [registry.py](../../rlinf/algorithms/registry.py) 注册函数解析。

### 训练 schedule

```yaml
rollout_epoch: 1            # 每 epoch 1 批 rollout
eval_rollout_epoch: 1
update_epoch: 4             # 每批 rollout 训 4 次
```

PPO 核心是数据重用。太大（>10）会 off-policy 漂得离谱。

### 三个 _type（重要）

```yaml
reward_type: chunk_level
logprob_type: chunk_level
entropy_type: token_level
```

- `reward_type`：env.step 执行整 50 步 chunk 累计成一个 scalar；最终 reward `[B]`
- `logprob_type`：flow_sde 是一次联合 Gaussian 采样 `[T=50, A=6]=300` 个值从同一 `(mean, std)` 抽。这次"决策"=整 300 值。PPO ratio `exp(Σ(new - old))`，sum 成 `[B]`。错配 `token_level` 把 300 当 300 个独立决策 clip，违背 flow_sde 语义
- `entropy_type: token_level`：正则项不进 ratio，保留 `[B, 50, 6]` 平均到每 (step, dim) 信号最细

详见 [08](08-代码改动清单.md) 末尾。

### 其它

```yaml
loss_agg_func: "token-mean"  # [B, T*A] loss reduce 方式；比 sample-mean 稳
bootstrap_type: always       # GAE 末尾 V(s_T) 总是 bootstrap

kl_beta: 0.0                 # KL penalty 系数（漂得快调 1e-3）
entropy_bonus: 0.005
clip_ratio_high: 0.2         # PPO ratio 上界
clip_ratio_low: 0.2
clip_ratio_c: 3.0            # 双向 clip（PPO-2 变体）
value_clip: 0.2
huber_delta: 10.0            # value loss Huber delta（10 几乎是 MSE）

gamma: 0.99
gae_lambda: 0.95

filter_rewards: False        # True 时钳到 [lower, upper]

sampling_params:
  do_sample: True
  temperature_train: 1.0
  temperature_eval: 0.0      # greedy，eval 可复现
  top_k: 0
  top_p: 1.0

length_params:
  max_new_token: null        # flow VLA 不生成 token
  max_length: 1024
  min_length: 1
```

`length_params` 是历史包袱给 token-AR 用，flow VLA 设 null 但字段必须存在。

## env

```yaml
env:
  channel:
    queue_size: 0           # 0 = 无 buffer 直通
  enable_offload: False

  train:
    total_num_envs: 8
    auto_reset: True
    max_episode_steps: 200
    max_steps_per_rollout_epoch: 200

  eval:
    total_num_envs: 4
    auto_reset: True
    ignore_terminations: True   # eval 跑满 200 步
    use_fixed_reset_state_ids: True   # 固定起始 state，可复现
    is_eval: True
```

`channel.queue_size: 0` 表无 buffer，env 必须等 actor 消费完才能产下一条。

`ignore_terminations: True` 让 eval episode 跑满（即使失败），fair 比较所有 trajectory。

## rollout

```yaml
rollout:
  mode: "colocate"
  generation_backend: "huggingface"
  recompute_logprobs: False
  enable_offload: True          # actor update 时 offload 到 CPU
  gpu_memory_utilization: 0.40
  model:
    ${actor.model}              # Hydra 引用，共享 actor.model 配置
```

`colocate` + `enable_offload: True` 是单卡标配。

`generation_backend: huggingface` 直接调 `forward()`；切 SGLang 改 `sglang`，但 SmolVLA 暂不支持。

`gpu_memory_utilization: 0.40` 是 rollout 显存上限（actor 拿剩 60%）。

## actor

### 基本

```yaml
actor:
  channel.queue_size: 0
  training_backend: "fsdp"
  micro_batch_size: 1           # 3090 硬限制
  global_batch_size: 8          # = total_num_envs
  seed: 42
  enable_offload: True          # rollout 时 offload
```

`micro=1 × accumulation=global/micro=8 → 8 sample/update step`，与 8 env 对齐。

### 模型

```yaml
model:
  with_value_head: True
  add_value_head: True
```

两个 True 触发 `SmolVLAForRLActionPrediction` 建 `Linear value_head`。

### Optimizer

```yaml
optim:
  type: adamw
  lr: 5.0e-6                  # LoRA（base 冻结）
  value_lr: 1.0e-4            # value head（新 init）
  weight_decay: 0.0           # LoRA 已低秩约束，加 WD 容易拉回 0
  adam_beta1: 0.9
  adam_beta2: 0.95            # 比 0.999 小，适应 RL noisy gradient
  adam_eps: 1.0e-5
  clip_grad: 1.0
  critic_warmup_steps: 0      # value 单独 warmup（震荡时设 50）
```

actor/value 分组：actor lr 小避免破坏 SFT prior；value lr 大让 critic 跟上。

### FSDP

```yaml
fsdp_config:
  strategy: "fsdp"
  sharding_strategy: "no_shard"   # 单卡不 shard
  gradient_checkpointing: True
  use_orig_params: True           # LoRA 必须
  mixed_precision:
    param_dtype: bf16
    reduce_dtype: fp32            # gradient all-reduce 用 fp32 更稳
    buffer_dtype: fp32
```

详见 [11 #2, #3, #9, #10](11-踩坑总结.md)。

## reward / critic —— 都关

```yaml
reward.use_reward_model: False    # 用 env dense reward，不训独立 reward model
critic.use_critic_model: False    # value head 内嵌 actor，不独立 critic worker
```

`use_reward_model: True` 适用 RLHF；`use_critic_model: True` 单独起 critic worker group 大模型分布式用。

## 重要依赖

| 字段 | 依赖 | 关系 |
|---|---|---|
| `actor.global_batch_size` | `env.train.total_num_envs` | ≤ envs 数 |
| `actor.micro_batch_size` | `global_batch_size` | 必须整除 |
| `actor.model.with_value_head` | `algorithm.loss_type` | `actor_critic` 时必 True |
| `rollout.gpu_memory_utilization` | actor 显存预算 | 加起来 < 0.95 |
| `algorithm.logprob_type` | 模型类型 | flow VLA 必 `chunk_level` |
| `length_params.max_new_token` | 模型类型 | flow VLA 必 `null` |
| `env.train.max_episode_steps` | `task.max_episode_steps` 和 `chunk_size` | 一致或 ≤，且整除 chunk_size |
