# 02 · RL 后训练路线

目标：在 SmolVLA LoRA SFT ckpt 上做 PPO RL 后训练，提升 SO-101 真机 pick-and-place 成功率。实施细节看 [08](08-代码改动清单.md)。

## 硬件能不能跑

**SmolVLA 卡线可行；OpenPI 仅 LoRA 紧张；OpenVLA-OFT 跑不动**。

| 模型 | 参数 | 3090 / 22 GB |
|---|---|---|
| SmolVLA-base | 450M | ✓ 卡线：8-16 env + grad ckpt + offload，推荐 LoRA |
| OpenPI π₀-base | 3B | 仅 LoRA：4-8 env，吞吐低 |
| OpenVLA-OFT | 7B | ✗ |

SmolVLA LoRA RL 显存预算合计 ~10-15 GB（actor 1 + base 1 + rollout 1-2 + sim 2-4 + activation 3-5 + overhead 2）。

## 为什么 LoRA RL

RL 阶段也只更新 LoRA，base 冻结：

- 不破坏 SFT 学到的能力，降低训崩风险
- 优化器状态从 ~6 GB 降到 ~0.5 GB，多塞 env
- 训完产物是 LoRA delta，真机直接 swap

⚠️ **`lora_alpha` 必须跟 SFT 一致**。LoRA scale = alpha / rank，alpha 改了等于权重瞬变，SFT 能力全打乱。

## 3090 必备配置

```yaml
cluster:
  component_placement:
    actor,env,rollout: 0-0          # collocated

env.train:
  total_num_envs: 8                  # 起步 8，稳了 16

rollout:
  gpu_memory_utilization: 0.40
  enable_offload: True

actor:
  micro_batch_size: 1
  global_batch_size: 8               # = total_num_envs
  enable_offload: True
  fsdp_config:
    gradient_checkpointing: True
  model:
    is_lora: True
    train_lora_only: True
```

## 路线

```
Phase 0  环境就绪          ✓ [01]
Phase 1  跑通 LIBERO 官方  跳过（3090 跑不动 7B）
Phase 2  跑通 ManiSkill3   [03]
Phase 3  SmolVLA 接入      ✓ [04/08]
Phase 4  自定义 pick_place ✓ [05/08]
Phase 5  开炼              ← 当前
```

Phase 5 重点看：`rollout/env_info/success_once`、`train/policy_loss` 不爆、`train/kl` 不漂、GPU 利用率 80-90%。

Phase 6 sim2real 常见手段：domain randomization、失败 case 补 SFT、真机做 RL。

- SmolVLA 接入卡住 → OpenPI（RLinf first-class，模板齐全），见 [06](06-训练Recipe.md)
