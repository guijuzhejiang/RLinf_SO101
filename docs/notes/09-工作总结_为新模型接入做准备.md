# 09 · 为新模型接入做准备

复盘"接一个自己 SFT 的 VLA 到 RLinf 跑 PPO"的 5 步通用流程。下次换 π0.5 / GR00T / OpenVLA-OFT 照走。

## 架构

```
用户侧
  examples/embodiment/config/<task>.yaml          总配方
                            /env/<env>.yaml      环境

模型层（每个模型一份）
  rlinf/models/embodiment/<name>/
    ├── __init__.py        get_model(cfg) → wrapper
    └── <name>_action_model.py   核心 wrapper（log_prob, sample, value, forward）

环境层（每个仿真器一份）
  rlinf/envs/<sim>/
    ├── agents/<robot>.py     注册机械臂
    └── tasks/<task>.py       reward / 成功判定

注册层（dispatch 改两处）
  rlinf/config.py            SupportedModel + 校验
  rlinf/models/__init__.py   builder 字典
  rlinf/envs/action_utils.py prepare_actions 分支（按需）
```

## 5 步

### 1. 配置层立起来

```python
# rlinf/config.py
class SupportedModel(Enum):
    ...
    YOUR_MODEL = "your_model"      # 加这一行
# validate_cfg 同步加分支

# rlinf/models/__init__.py
def _build_your_model(cfg, torch_dtype=None):
    from rlinf.models.embodiment.your_model import get_model
    return get_model(cfg, torch_dtype)
_MODEL_BUILDERS["your_model"] = _build_your_model
```

### 2. 写模型 wrapper（核心，95% 工作量）

`rlinf/models/embodiment/<name>/<name>_action_model.py` 必须实现：

| 方法 | 输入 | 输出 | 何时 |
|---|---|---|---|
| `predict_action_batch(obs)` | obs dict | `{actions, prev_logprobs, prev_values, ...}` | rollout |
| `default_forward(obs, actions)` | obs + actions | `{logprobs, values, entropy}` | actor update |
| `gradient_checkpointing_enable/disable()` | — | — | FSDP wrap |

**dtype 契约**：出口 `logprobs/values/entropy` 必须 `float32`（PPO loss 有 assert），内部可全 bf16，return 前 `.float()`。

flow VLA 关键三件事：
1. **一次性 SDE 注入采样**：把 `[B, T, A]` noise 在某一步换成 `eps` + Gaussian，记 mean/std
2. **chunk-level log p**：`[B, T, A]` 在 `(T,A)` 上 sum → `[B]`
3. **value head 接 suffix 输出**：expert 最后 hidden state，简单 MLP 即可

抄 `smolvla_action_model.py` 或 `openpi_action_model.py` 改名字 / shape。

### 3. 写仿真环境（可选）

跑现成 ManiSkill / LIBERO / Calvin 跳过。自定义任务需要：

| 文件 | 干嘛 |
|---|---|
| `agents/<robot>.py` | 注册机械臂，关节名 / 相机 / 控制器 |
| `tasks/<task>.py` | 场景、`reset()`、`evaluate()`（reward + success） |
| `agents/__init__.py` | `import` 触发注册 |

注意：
- 相机命名按 `_wrap_obs` 硬约定：`simple` 模式找 `base_camera`，`default` 找 `3rd_view_camera`
- `evaluate()` info 字段要么对齐 RLinf 默认 reward（`is_src_obj_grasped` / `consecutive_grasp`），要么 `reward_mode: "raw"` 走自己的 dense reward

### 4. 控制空间对接

`rlinf/envs/action_utils.py` 的 `prepare_actions_for_<sim>` 是 model↔sim 胶水。默认是 EE 控制（world_vector + rot_axangle + gripper）。joint-pos / joint-vel 输出加 bypass：

```python
def prepare_actions_for_maniskill(raw, ...):
    if "panda" in policy:  return raw
    if "so101" in policy:  return raw
    ...  # 下面是 EE 解析
```

`model/<name>.yaml` 同步设 `policy_class` / `control_mode`。

### 5. 配置文件三件套

`examples/embodiment/config/<task>_<algo>_<model>_<robot>.yaml`：

```yaml
defaults:
  - env/<env_yaml>@env.train
  - env/<env_yaml>@env.eval
  - model/<model_yaml>@actor.model
  - training_backend/fsdp@actor.fsdp_config

algorithm:
  adv_type: gae
  loss_type: actor_critic
  reward_type:  chunk_level
  logprob_type: chunk_level    # flow VLA 必须
  entropy_type: token_level
  length_params:
    max_new_token: null        # flow VLA 不是 autoregressive
    max_length: 1024

actor:
  micro_batch_size: 1
  global_batch_size: <= total_num_envs
  model:
    with_value_head: True
    add_value_head: True
  fsdp_config:
    use_orig_params: True      # LoRA 必须
    mixed_precision:
      param_dtype: bf16        # RLinf 自己的命名，不是 bfloat16
      reduce_dtype: fp32
      buffer_dtype: fp32
```
