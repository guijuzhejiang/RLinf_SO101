# 16 · 训练时进程与显存分布

`nvidia-smi` 看 RLinf 训练时 GPU 三个常驻进程。

## 实测快照（SmolVLA + SO101，3090 单卡 collocated）

```
PID         Type   Process name                          GPU Memory
2141884      C     ...EmbodiedFSDPActor.run_training       5314MiB
2141886      C     ray::MultiStepRolloutWorker              476MiB
2141888    C+G     ray::EnvWorker                          8070MiB
                                                  合计 ≈ 13.5 GB
```

三进程对应 RLinf 训练循环三角色：

```
                ┌─────────────────┐
          obs   │ EnvWorker (8 GB)│  ← 仿真器 + GPU 渲染
   ┌────────────┤ ManiSkill GPU   ├────────────┐
   │            │ sim + 3 相机    │            │
   ▼            └─────────────────┘            │
┌──────────────┐                       ┌───────────────────┐
│ Rollout      │   action              │ Actor             │
│ (0.5 GB)     ├──────────────────────►│ (5 GB)            │
│ HF 推理      │                       │ FSDP + LoRA train │
│ 模型 offload │◄── sync weights ──────│ + optimizer       │
└──────────────┘                       └───────────────────┘
```

## 1. `EmbodiedFSDPActor` — 5314 MiB

**训练 worker**。攒够一批 rollout 后跑 PPO loss + backward + opt step。源码 [fsdp_actor_worker.py:972](../../rlinf/workers/actor/fsdp_actor_worker.py#L972)。

**构成（~5.3 GB）**：

| 组件 | 大小 | 备注 |
|---|---|---|
| bf16 base 模型（FSDP） | ~900 MB | SmolVLA 450M × 2 bytes |
| LoRA adapter | ~3 MB | rank=32, 1.5M params bf16 |
| AdamW opt state | ~24 MB | LoRA params fp32 m + v |
| FSDP shard buffer | ~50 MB | no_shard 单卡 |
| 前向 activation（bf16） | ~2-3 GB | bs=1 × chunk=50 × SmolVLM2 16 层，grad_ckpt 已打折 |
| backward grad buffer | ~1 GB | 临时 |
| CUDA context + workspace | ~500 MB | cuBLAS / cuDNN |

**行为**：`enable_offload: True`，rollout 阶段被 offload 到 CPU → < 500 MB。截图是 actor 正在 train。

**调优**：
- 显存吃紧：`gradient_checkpointing: True`（已开）；降 `micro_batch_size`（已 1）
- 想跑更大 batch：上 2 卡分 actor / rollout，或关 `gradient_checkpointing` 换 2× 速度

## 2. `ray::MultiStepRolloutWorker` — 476 MiB

**推理 worker**。obs → `predict_action_batch` → action chunk → env，循环跑完 trajectory。同时记 `prev_logprobs / prev_values`。源码 [huggingface_worker.py:36](../../rlinf/workers/rollout/hf/huggingface_worker.py#L36)。

**只占 476 MB 的原因**：模型也是 SmolVLA 450M，正常推理 1-2 GB。但 yaml `enable_offload: True`，actor train 时 rollout 模型权重 offload 到 CPU，GPU 只剩 CUDA context、KV cache 元数据、几个常驻 buffer。rollout 阶段截图会涨到 1.5-2 GB。

**行为**：collocated 模式，actor / rollout 互相 offload。模型权重 RAM ↔ GPU 来回搬运是单卡训练的最大开销。

**调优**：
- 嫌 offload 慢：2 卡 `actor: 0-0, rollout: 0-1`
- 想再快：SGLang，但要支持 SmolVLA model runner

## 3. `ray::EnvWorker` — 8070 MiB（`C+G`）

**仿真环境 worker**。ManiSkill GPU 物理 + 3 路相机渲染。源码 [env_worker.py:47](../../rlinf/workers/env/env_worker.py#L47)。

**`C+G` 含义**：`C`=Compute（CUDA），`G`=Graphics（OpenGL/Vulkan）。**只有这进程同时跑两种 API**：
- C：SAPIEN 物理引擎算 8 env 刚体动力学、碰撞检测
- G：每 env 渲染 base_camera (640×480) + wrist_camera (640×480) + render_camera (1024×768) 三路 RGB，Vulkan rasterizer

**构成（~8 GB）**：

| 组件 | 大小 |
|---|---|
| Vulkan/GL context | ~1-2 GB |
| 物理 sim GPU state | ~500 MB |
| 渲染 framebuffer | ~1 GB |
| Mesh/texture 缓存 | ~2 GB |
| Shader/pipeline cache | ~500 MB |
| 其他/碎片 | ~2 GB |

8 GB 在 8 env + 3 camera 这规模属正常偏高（典型 4-10 GB）。

**行为**：`enable_offload: False`，物理 + 渲染 context 初始化后无法热 offload，一直占着。reset 时峰值（重建场景）。

**调优**：
- 最有效：减 env 数。`total_num_envs: 8→4` 砍一半
- 砍相机分辨率 640×480→256×256 省 ~1 GB
- 关 render_camera（只录视频时需）：`render_mode: "rgb_array" → "sensors"` 省 ~500 MB
- 关 wrist_camera：精度会掉，不推荐
- `sim_backend: "cpu"`：speed 断崖

## 总账

| 进程 | 当前 | rollout 时 | actor train 时 | 峰值 |
|---|---|---|---|---|
| Actor | 5.3 GB | <0.5 GB | 5.3 GB | 7 GB |
| Rollout | 0.5 GB | 1.5-2 GB | 0.5 GB | 2 GB |
| EnvWorker | 8 GB | 8 GB | 8 GB | 10 GB |
| **合计** | **13.5 GB** | **9.5 GB** | **13.5 GB** | **19 GB** |

3090 可用 22 GB，剩 2-3 GB margin。**继续放大 batch 或加 env 就 OOM**。

## 单卡 vs 多卡

| 部署 | actor | rollout | env | 总 | 速度 |
|---|---|---|---|---|---|
| 3090 collocated（当前） | offload 切换 | offload 切换 | 一直 | 13-19 GB | 慢，actor↔rollout 串行 |
| 2 卡分卡 | 独占 GPU0 | 独占 GPU1 | 跟 actor 共 GPU0 | GPU0:13G / GPU1:2G | actor train 时 rollout 可并行 |
| H100 80G 单卡 | 一直 | 一直 | 一直 | 13-15 GB | 不 offload，最快 |

详见 [14](14-多GPU训大模型.md)。
