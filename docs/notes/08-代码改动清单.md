### `models/embodiment/smolvla/smolvla_action_model.py`

核心 wrapper。flow_sde 思路：在 num_steps 步反推中随机挑一步注入 Gaussian noise，让这一步变成 Gaussian transition，log p 用正态 PDF 算；其余步走 ODE 保持跟 SFT 一致。

```
v_t        = action_out_proj(suffix_out)
x0_pred    = x_t - v_t * t
x1_pred    = x_t + v_t * (1 - t)
sigma      = noise_level * sqrt(t / (1-t))
flow_sde:  mean = x0_pred*(1-(t-dt)) + x1_pred*(t-dt - sigma²*dt/(2t))
           std  = sqrt(dt) * sigma
x_next     = mean + N(0,1) * std
log p      = -log(σ) - 0.5*log(2π) - 0.5*((x_next - mean)/σ)²
```

实现要点：
- **PEFT 穿透**：`_base_policy / _flow_model / _policy_config` property 不管外面是 PEFT/FSDP/裸模型都能拿底层
- **`_get_velocity`** 内联 SmolVLA `denoise_step`，额外返回 `suffix_out`（value head 输入）
- **`sample_actions`** rollout 跑完整 10 步，挑一步注入 noise，每步算 value，取注入步的 logprob + value 作为 `prev_logprobs / prev_values`
- **`default_forward`** 训练时用 rollout 存的 `chains + denoise_inds + obs` 重算 mean/std/log p/value
- **ValueHead**：多层 MLP（参考 OpenVLA），输入 `suffix_out` 沿 chunk 维 mean-pool，输出 `[B]`；activation `relu`（kaiming_normal_ 不支持 gelu）

**dtype 坑**：SmolVLA 内部强制把 `suffix_out` upcast fp32 再过 `action_out_proj`，模型整体不能直接 `.to(bfloat16)`，会 dtype mismatch。训练省显存只能 `torch.autocast(dtype=bfloat16)`。

## PPO 三个 `*_type` 字段

控制 rollout 数据做 PPO loss 时聚合的粒度。三字段独立但要配套。

VLA 一次 forward 产生：
```
actions/logprobs: [B, 50, 6]
rewards:          [B, 50]
```

| `_type` | 处理 | shape | 含义 |
|---|---|---|---|
| `token_level` | 不动 | `[B, 50, 6]` | 每个 (step, dim) 一个 |
| `action_level` | sum 掉 dim | `[B, 50]` | 每个时间步一个 |
| `chunk_level` | sum 掉 step+dim | `[B]` | 整 chunk 一个 |

**按模型采样语义选**：

| 模型 | 采样方式 | logprob_type |
|---|---|---|
| flow VLA（SmolVLA / OpenPI / GR00T） | 一次出整个 chunk，是联合 Gaussian | **`chunk_level`** |
| 离散 token VLA（OpenVLA-OFT） | 自回归一 token 一 token | `token_level` |

SmolVLA flow `[50,6]=300` 个值是同一 `(mean, std)` 一次抽出来的 → 一次"动作选择"= 一个 PPO ratio → `chunk_level`。`token_level` 把它当 300 个独立决策做 clip 违反 flow_sde 设计。

**最终设置**：

```yaml
reward_type:  chunk_level    # env 给 chunk 一个累计 reward
logprob_type: chunk_level    # flow 一次联合采样 = 一次决策
entropy_type: token_level    # entropy 是正则，细粒度好
```

跟 OpenPI π0.5 / GR00T / DexBotic / StarVLA 40+ 个生产 yaml 一致。

**常见误解**：选 `chunk_level` 不会丢 6 关节信息 —— 6 关节**采样值**完整保留在 `actions [B,50,6]` 送给 env；`chunk_level` 只决定**log p 怎么 sum 起来算 ratio**。

## 关键设计决策

| 决策 | 选择 | 原因 |
|---|---|---|
| 算法 | PPO | 长视距 + 多阶段稠密 reward；value head + GAE 比 GRPO baseline 稳；LoRA RL 下 value head VRAM 可忽略 |
| Flow log p | flow_sde | 实现成本最低，OpenPI 已验证；纯数学等价 |
| logprob_type | chunk_level | flow 一次联合采样 = 一次决策 |
| reward_type | chunk_level | env 在 chunk 内累计 reward 一次给 |
| entropy_type | token_level | 正则项不进 ratio，细粒度更准 |
| ValueHead | 多层 MLP | 参考 OpenVLA，单 Linear 表达力不足 |
| ValueHead activation | relu | kaiming_normal_ 不支持 gelu |
| Value 池化 | chunk 维 mean-pool | 参考 OpenPI；SmolVLA 无 "action token" 概念 |
| 机械臂颜色 | 生成 MJCF 副本 | 不动原文件；hash 缓存 |
| 盘子方向 | initial_pose quat | 一行解决 |
| Wrist 相机 | sapien look_at + -Y 偏移 | local frame 翻转 + 电机遮挡 |
| dtype | fp32 + autocast | SmolVLA 强制 fp32 走 `action_out_proj` |
